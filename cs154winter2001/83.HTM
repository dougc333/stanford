P is closed under union.  For any two P-languages L1 and L2 let M1 and
M2 be the TM's that decided them in polynomial time.  We construct a TM M'
that decides the union of L1 and L2 in polynomial time:

M'="On input <w>:
1. Run M1 on w. If it accepts, accept.
2. Run M2 on w. if it accepts, accept. Otherwise reject."

M' accepts w iff either M1 or M2 ACCEPTS w. Therefore, M' decides the union
of L1 and L2.  Since both stages take polynomial time, the algorithm runs
in polynomial time.

P is closed  under Concatenation.  For any two P-languages L1 and L2 let
M1 and M2 be the TM's that decide them in polynomial time.  We construct a
TM M' that decides the Concatenation of L1 and L2 in polynomial time:
M'="On input <w>:
1.  For each i from 0 to (length(w) – 1)

2.  w as a string of characters, w1w2….wn.  Split w into wA = w1w2…wi and wB = w(i+1)…wn.
3. Run M1 on wA and M2 on wB.  If both accept, accept.
4. If w is not accepted after trying all the possible cuts reject."
M' accepts w iff w can be written as wAwB such that M1 accepts wA and M2
accepts wB.  Therefore, M' decides the concatenation of L1 and L2.  Since
stage 2 runs in polynomial time and is repeated at most 0(n) times, The
algorithm runs in polynomial time.

 

P is closed under complement.  For any P-languages L, let M be the TM
that decides it in polynomial time.  We construct a TM M' that decides the
complement of L in polynomial time:
 
M'="On input <w>:
1. Run M on w.
2. If M accepts, rejects. If it rejects, accept."
M' decides the complement of L. Since M runs in polynomial time, M' also
runs in polynomial time.
