Consistent Distributed Shared Memory

CS244b Assignment #1
Instructor: David Cheriton
Assigned: Tuesday, March 30, 1999
Design Due: Monday, April 12, 1999
Due: Friday, April 23, 1999

We estimate that this assignment will require about 25 hours of work, so please plan ahead!

1. Introduction

The purpose of this assignment is to get you to understand the key notions of shared memory consistency and distributed interprocess synchronization, protocols for implementing these, and the performance, reliability and usability issues with distributed shared memory.

To this end, you are to extend a simple distributed shared memory system that can be used with simple applications running in parallel across at least three machines, and collect some basic performance measurements from running this software.

2. Starting Point

The relevant distributed shared memory issues, techniques and protocols are discussed in Chapter 2 of the reader and in the Li/Hudak and TreadMarks papers.

You are provided with a simple C++ implementation of a DSM, as well as some sample applications (see /usr/class/cs244b/shared-memory). Your first objective should be to familiarize yourself with the code, run the sample applications, and become comfortable with the workings of the system.

A tutorial is scheduled for Friday, April 2 (11:00-11:50) to review the assignment, discuss the provided code and answer questions. You should read the relevant material and look over the given code in advance of this tutorial.

3. What to submit

The first deadline is for submission of the design for your modifications. It is sufficient to submit a state diagram for the client and server.

For the final deadline, you must submit the following:

  1. The final design for your modifications. You should justify any changes that you made from your initial design submission.
  2. The code for each modification. You should clearly indicate your modifications in the code. You should also supply a README file for each modification that outlines the code changes that you made for that modification.
  3. For each modification (where appropriate), a concise description of the consistency and syncronization protocols that you implemented, and how they interact.
  4. A description of the performance of your distributed shared memory system and applications, including:
    The time for a memory reference for which the page needs to be fetched from the server,
    The time for a memory reference when the page is owned by another process,
    The elapsed time for running the applications using this shared memory entirely locally, across 2 machines and across 3 machines (or more if you so choose).
  5. Your recommendation of when to use distributed shared memory, any caveats on its use, and the effect of different page sizes, network speeds, memory sizes and processor speeds.

In addition, you will receive a short list of design questions by email a few days after you submit your assignment. These questions will require you to write a short paragraph about how you solved two or three key design issues. You will also have to give references to the locations in the code where these solutions are implemented. The entire solution to these design questions needn't exceed a page in length. You will have 48 hours from the receipt of these questions to email back your answers.
